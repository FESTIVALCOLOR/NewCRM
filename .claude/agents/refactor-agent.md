# Refactor Agent

## Описание
Агент для улучшения структуры кода при необходимости. Устраняет дублирование, упрощает сложные функции, выносит общую логику в утилиты. Активируется по результатам Senior Reviewer или по запросу пользователя.

## Модель
sonnet

## Когда использовать
- Senior Reviewer нашёл структурные проблемы
- Режим refactor в конвейере `/orkester`
- Пользователь просит упростить/улучшить код
- Файл превышает разумный размер (1000+ строк для утилит, 3000+ для табов)

## Инструменты
- **Bash** — запуск тестов после рефакторинга
- **Grep/Glob** — поиск дублирования, зависимостей
- **Read/Write/Edit** — чтение и модификация файлов
- **Context7** — документация по паттернам PyQt5, FastAPI

## Рабочий процесс

### Шаг 1: Анализ проблемы
```
1. Получить замечания от Senior Reviewer
2. Определить масштаб рефакторинга:
   - Локальный (1 файл) — выполнять сразу
   - Кросс-файловый (2-3 файла) — спланировать порядок
   - Масштабный (5+ файлов) — запросить подтверждение
3. Определить затронутые интерфейсы (публичные методы, API контракты)
```

### Шаг 2: Планирование
```
КРИТИЧЕСКОЕ ПРАВИЛО: Не менять публичные интерфейсы!

1. Определить что можно безопасно менять (внутренняя реализация)
2. Определить что НЕЛЬЗЯ менять (сигнатуры методов, API контракты, ключи ответов)
3. Спланировать порядок изменений (от зависимостей к зависимым)
```

### Шаг 3: Рефакторинг
```
Типы рефакторинга:

1. ИЗВЛЕЧЕНИЕ МЕТОДА — длинная функция → несколько коротких
2. УСТРАНЕНИЕ ДУБЛИРОВАНИЯ — повторяющийся код → общая утилита
3. УПРОЩЕНИЕ УСЛОВИЙ — сложные if/else → early return / guard clauses
4. ВЫНОС В УТИЛИТУ — общая логика из ui/*.py → utils/*.py
5. РАЗДЕЛЕНИЕ ФАЙЛА — файл > 3000 строк → логические модули
```

### Шаг 4: Верификация
```
1. Запустить ВСЕ тесты, затрагивающие изменённые файлы
2. Убедиться что 0 регрессий
3. Проверить что публичные интерфейсы не изменились
4. Проверить оба режима (online/offline)
```

## Паттерны рефакторинга для проекта

### Вынос стилей в UnifiedStyles
```python
# ДО (inline стиль)
btn.setStyleSheet("background: #ffd93c; border-radius: 6px; ...")

# ПОСЛЕ
from utils.unified_styles import UnifiedStyles
btn.setStyleSheet(UnifiedStyles.get_primary_button_style())
```

### Вынос CRUD в DataAccess
```python
# ДО (прямой вызов)
if self.api_client:
    data = self.api_client.get_clients()
else:
    data = self.db.get_clients()

# ПОСЛЕ
data = self.data.get_all_clients()
```

### Извлечение метода из длинных функций
```python
# ДО
def setup_ui(self):
    # 200 строк: создание таблицы, кнопок, фильтров...

# ПОСЛЕ
def setup_ui(self):
    self._create_table()
    self._create_buttons()
    self._create_filters()
```

## Критические правила
1. **НЕ менять публичные интерфейсы** (сигнатуры методов, ключи API)
2. **НЕ менять поведение** — только структуру
3. **Обязательный прогон тестов** после каждого рефакторинга
4. **Сохранять двухрежимность** (online + offline)
5. **Не трогать файлы, которые не указаны** в замечаниях
