# Фичи и спецификации

> Бизнес-логика, Kanban, платежи, таймлайны, Яндекс.Диск, offline-режим.

## Двухрежимная архитектура

### Сетевой режим (Data Flow)

```
LoginWindow → APIClient.authenticate() → (JWT токен + employee_data)
    → MainWindow + SyncManager (QTimer 30 сек)
    → Табы работают через DataAccess (API + fallback на SQLite)
    → При изменении: DataAccess → API + offline очередь
```

### Автономный режим (Data Flow)

```
LoginWindow → DatabaseManager.verify_credentials() (SQLite)
    → MainWindow (без SyncManager)
    → Табы работают через DataAccess (только SQLite)
```

### Drag & Drop

Карточки перемещаются между колонками через Drag & Drop. При перемещении:
1. Обновляется `column_name` карточки
2. Пересчитывается `order_position`
3. Синхронизация с сервером

## Workflow — рабочий процесс стадии

```
Назначение исполнителя (ExecutorSelectionDialog)
  → deadline = assigned_date + norm_days рабочих дней
  → stage_executor.completed = False
          │
          ▼
Исполнитель: "Сдать работу"
  → API: complete_stage_for_executor() + workflow_submit()
  → stage_executor.completed = True
  → Кнопка → "Ожидайте проверку" (disabled)
          │
          ▼
Менеджер: "Принять работу" / "На исправление"
  ├── "Принять" → workflow_accept() → дата в timeline
  └── "На исправление" → workflow_reject()
        → stage_executor.completed = False (сброс)
        → Исполнитель снова видит "Сдать работу"
          │
          ▼
"Клиенту на согласование" → workflow/client-send
  → "Клиент согласовал" → workflow/client-ok
```

### DB модель StageWorkflowState

```sql
status: 'in_progress' | 'revision' | 'client_approval' | 'completed'
revision_count: Integer   -- кол-во исправлений
```

### Расчёт norm_days

```python
K = max(0, int((area - 1) // 100))  # коэффициент площади
raw_norm_days = базовые_дни(K)       # для каждого этапа
norm_days = contract_term * (raw_norm_days / sum(in_scope_raw))  # пропорционально
```

### Расчёт actual_days

```python
actual_days = networkdays(prev_date, current_date)  # рабочие дни (Пн-Пт)
```

### Авто-расчёт START

```python
START = max(contract_date, survey_date, tech_task_date)
```


### Автоматические операции

| Событие | Действие |
|---------|----------|
| Создание договора | Создаётся папка на Я.Диске |
| Изменение типа/города/адреса | Папка перемещается |
| Удаление договора | Папка удаляется |
| Загрузка файла | Файл → Я.Диск → БД запись |

### Ограничения

- Максимальный размер файла: **200 MB**
- Таймаут загрузки: `max(60, size_mb * 2 + 60)` секунд
- Retry: 3 попытки с backoff (1, 2, 4 сек)

## Offline режим

### Последовательность

```
1. Приложение пытается API запрос
2. При ошибке → fallback на локальную SQLite
3. Операции записи → очередь (offline_manager)
4. При восстановлении → синхронизация очереди
```

### Поддерживаемые типы синхронизации

- client, contract, crm_card, supervision_card
- employee, payment, yandex_folder, project_file

### Таймауты

| Параметр | Значение |
|----------|----------|
| DEFAULT_TIMEOUT | 10 сек |
| WRITE_TIMEOUT | 15 сек |
| OFFLINE_CACHE_DURATION | 10 сек |
| CONNECTION_CHECK | 60 сек |
| PING_TIMEOUT | 2 сек |
