## 1. Два вида уведомлений

| | Вид 1: Личные уведомления | Вид 2: Групповой чат |
|---|---|---|
| **Кому** | Конкретному сотруднику | Клиенту + команде проекта |
| **Куда** | Личный Telegram сотрудника | Telegram-группа проекта |
| **О чём** | Внутренние события CRM (назначение, дедлайн, смена стадии) | Ход проекта для клиента (скрипты по стадиям) |
| **Когда** | Автоматически при событии | При нажатии кнопки или автоматически при смене стадии |
| **Файлы** | Нет | Памятки (PDF), файлы стадии, отчёты |

**Три отдельных чата уведомлений:**
- **Индивидуальные проекты** — свой набор уведомлений и скриптов
- **Шаблонные проекты** — свой набор уведомлений и скриптов
- **Авторский надзор** — свой набор уведомлений и скриптов

## 2. Личные уведомления — Индивидуальные проекты

### Типы событий

| Тип | Настройка | По умолч. | Описание |
|-----|-----------|-----------|----------|
| `assigned` | `notify_assigned` | Вкл | Назначение на карточку |
| `crm_stage_change` | `notify_crm_stage` | Вкл | Сдача, проверка, согласование, акт |
| `deadline` | `notify_deadline` | Вкл | Предупреждение о дедлайне (за 2 дня) |
| `payment` | `notify_payment` | Выкл | Создание/изменение оплаты |

## 5. Правила дублирования уведомлений

### Правило 1: Старший менеджер → Руководитель студии + Менеджер
Все уведомления, получаемые старшим менеджером, **также получает руководитель студии и менеджер** (если он назначен на карточку).

### Правило 2: СДП/ГАП → Старший менеджер (без текста действия)
Все уведомления, отправляемые СДП или ГАП, **дублируются старшему менеджеру** в информационном виде — без фраз "Вы — проверяющий", "Проверьте" и подобных призывов к действию.

> Пример: СДП получает "Чертёжник сдал работу по проекту {address}. Проверьте." → Старший менеджер получает "Проект {address}: чертёжник сдал работу (Стадия 1)."

### Правило 3: Исправления исполнителям → Старший менеджер
Все уведомления об отправке на исправление, получаемые исполнителями (дизайнер, чертёжник), **также дублируются старшему менеджеру** (информационно: "Проект {address}: работа возвращена на исправление").

### Правило 4: Шаблонные — Менеджер/ГАП → Старший менеджер (без текста действия)
Для шаблонных проектов: все уведомления менеджеру или ГАП **дублируются старшему менеджеру** в информационном виде — без призывов к действию.

## 11. Переменные шаблонов

### Основные переменные

| Переменная | Что подставляется | Пример |
|-----------|-----------------|--------|
| `{client_first_name}` | **Имя клиента** (только имя, без фамилии и отчества) | Иван |
| `{client_name}` | Полное имя клиента (для внутренних уведомлений) | Иванов Иван Иванович |
| `{address}` | Адрес объекта (город входит в адрес) | Москва, Кутузовский 220 |
| `{contract_number}` | Номер договора | 001-0001 |
| `{area}` | Площадь (м²) | 150 |
| `{stage_name}` | Название текущей стадии | Стадия 1: Планировочное решение |
| `{deadline}` | Дедлайн / срок на рассмотрение | 20.03.2026 |
| `{deadline_date}` | Дата дедлайна (для внутренних уведомлений) | 20.03.2026 |
| `{stage_files}` | Ссылки на загруженные файлы стадии | Файлы: ссылка1, ссылка2 |
| `{revision_count}` | Номер правки | 2 |
| `{visit_date}` | Дата выезда надзора | 15.03.2026 |
| `{pause_reason}` | Причина приостановки | Ожидание материалов |
| `{amount}` | Сумма оплаты | 150 000 |
| `{review_link}` | Ссылка на страницу отзывов | https://... |

### Переменные команды проекта

| Переменная | Что подставляется | Пример |
|-----------|-----------------|--------|
| `{senior_manager}` | ФИО старшего менеджера | Сидоров Пётр Иванович |
| `{senior_manager_username}` | Telegram username старшего менеджера | petr_sidorov |
| `{manager_name}` | ФИО менеджера | Петрова Анна Сергеевна |
| `{manager_username}` | Telegram username менеджера | anna_petrova |
| `{sdp}` | ФИО СДП | Козлова Мария Александровна |
| `{sdp_username}` | Telegram username СДП | maria_kozlova |
| `{director}` | ФИО руководителя студии | Фёдоров Алексей Владимирович |
| `{director_username}` | Telegram username руководителя | alexey_fedorov |
| `{dan}` | ФИО ДАН | Новикова Елена Дмитриевна |
| `{dan_username}` | Telegram username ДАН | elena_novikova |

**Формат в тексте:** В начальных скриптах групповых чатов в блоке "Ваша команда" каждый сотрудник указывается в формате:
```
 — Роль: ФИО (@username)
```
Символ `@` перед username создаёт кликабельную ссылку на профиль Telegram.
