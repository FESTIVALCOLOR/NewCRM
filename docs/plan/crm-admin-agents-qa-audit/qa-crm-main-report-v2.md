## Новые проблемы

| ID | Серьёзность | Описание | Файл:строка |

| НБ1 | КРИТИЧЕСКАЯ | Дашборд Авторского надзора: `active_orders` всегда 0. Условия взаимоисключающие: `contract_condition = Contract.status == 'АВТОРСКИЙ НАДЗОР'` + `~Contract.status.in_(['СДАН', 'РАСТОРГНУТ', 'АВТОРСКИЙ НАДЗОР'])`. Пересечение — пустое множество. Результат: виджет "Активные заказы" в дашборде надзора всегда показывает 0. | `dashboard_router.py:172-195` |

| НН1 | НИЗКАЯ | `status_label` для "АВТОРСКИЙ НАДЗОР" в архивной карточке не имеет цветового стиля. `ArchiveCard.init_ui()` добавляет цветной лейбл только для СДАН (зелёный) и РАСТОРГНУТ (красный). Для АВТОРСКОГО НАДЗОРА лейбл отображается как белый/стандартный без синего фона, хотя рамка карточки синяя (#2196F3). | `crm_archive.py:172-192` |
| НН2 | НИЗКАЯ | `get_all_payments_optimized` не включает оклады (`Salary`). Если клиентский код использует этот эндпоинт для отображения всех выплат — оклады будут отсутствовать. Основной `GET /api/payments/` покрывает оба источника, но оптимизированный — только таблицу `payments`. | `payments_router.py:605-664` |
| НН3 | НИЗКАЯ | `complete_stage` (PATCH `stage-executor/{stage_name}`) находит запись по `stage_name` + `crm_card_id`, берёт последнюю по `id`. При нескольких назначениях на одну стадию разных исполнителей — обновляется только последняя запись.
