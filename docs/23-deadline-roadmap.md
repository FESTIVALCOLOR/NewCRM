```python
def get_current_substage(contract_id, stage_group):
    """Найти первый незаполненный подэтап в stage_group"""
    entries = get_timeline_entries(contract_id)
    for e in entries:
        if e.stage_group == stage_group and not e.actual_date:
            return e
    return None  # все заполнены
```

## 6. Архитектурные решения

### 6.1 Иерархия дедлайнов

```
Дедлайн проекта (crm_cards.deadline)
  = START + contract_term (рабочие дни)
  = авто-расчёт, ручное изменение через диалог с причиной

  └── Дедлайн исполнителя (stage_executors.deadline)
        = prev_actual_date + norm_days (рабочие дни)
        = авто-расчёт при назначении, изменение через диалог с причиной

        └── Плановые даты подэтапов (project_timeline_entries._planned_date)
              = prev_date + norm_days (цепочка)
              = авто-расчёт, только в tooltip
```

### 6.2 Потоки данных

```
Администрирование (NormDaysTemplate)
  ↓ (одностороннняя синхронизация)
Карточка CRM → Таблица сроков (norm_days)
  ↓ (при назначении исполнителя)
ExecutorSelectionDialog → дедлайн = prev_date + norm_days
  ↓ (если СДП изменил)
custom_norm_days → отображение: зачёркнуто + красным
  ↓ (workflow действия)
actual_date → зелёный фон в таблице
  ↓ (каскадный пересчёт)
все последующие planned_dates → tooltip обновлены
```

### 6.3 Правила синхронизации norm_days

1. **Направление:** Администрирование → Карточка (НИКОГДА наоборот)
2. **Триггер:** При загрузке карточки + при инициализации timeline
3. **Исключение:** Если СДП/ГАП/менеджер при назначении ставит бОльшую норму → сохраняется как custom_norm_days
4. **Приоритет шаблонов:** Конкретный агент > "Все агенты" > Формулы
5. **Перезапись:** Только записи с norm_days=0 обновляются из шаблона. Записи с norm_days>0 НЕ перезаписываются
