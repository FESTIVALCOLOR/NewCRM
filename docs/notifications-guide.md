## Как устроена система уведомлений

В CRM реализованы два независимых вида уведомлений — они решают разные задачи.

| | Вид 1: Личные уведомления | Вид 2: Групповой чат |
|---|---|---|
| Кому | Сотруднику | Клиенту + команде |
| Куда | Личный Telegram сотрудника | Telegram-группа проекта |
| О чём | События внутри CRM | Ход реализации проекта |
| Кто настраивает | Директор / сам сотрудник | Директор / менеджер |

---

## Вид 1 — Личные уведомления сотрудников

### Что это такое

Сотрудник получает **личное сообщение в Telegram** от CRM-бота, когда с ним происходит что-то важное в системе.

### Какие события доступны

| Событие | Описание |
|---------|----------|
| Назначение исполнителем | Сотрудника назначили ответственным за стадию проекта |
| Смена стадии CRM | Карточка проекта перешла в новую колонку |
| Предупреждение о дедлайне | Срок по стадии подходит к концу |
| Создание оплаты | По договору создана новая запись оплаты *(только для сотрудников с доступом к оплатам)* |
| Новый надзор | Появилась новая карточка в надзоре |

### Как это работает (схема)

```
Событие в CRM (смена стадии, назначение исполнителя...)
        |
        v
Система создаёт запись уведомления в базе данных
        |
        v
Проверяет: включён ли этот тип событий в настройках сотрудника?
        |
   Да  |  Нет
        |    \
        v     --> Уведомление сохранено в базе, но не отправлено
Отправляет личное сообщение в Telegram сотрудника
```

## Вид 2 — Групповой чат заказа

### Что это такое

Для каждого проекта (CRM-карточки) создаётся отдельная **Telegram-группа**, куда входят клиент и сотрудники команды. При переходе проекта между стадиями бот автоматически отправляет сообщения по заранее настроенным шаблонам.

### Какие события запускают сообщения

| Тип скрипта | Когда срабатывает |
|-------------|------------------|
| Начало проекта | Менеджер нажимает кнопку "Начальный скрипт" в карточке |
| Завершение стадии | Карточка переходит в следующую колонку |
| Завершение проекта | Карточка переходит в финальную стадию |
| Событие надзора | Надзорная карточка меняет стадию |

### Как это работает (схема)

```
Событие (смена стадии / кнопка скрипта в карточке)
        |
        v
Система находит активный групповой чат этой карточки
        |
        v
Находит включённый шаблон-скрипт нужного типа
        |
        v
Подставляет данные: имя клиента, адрес, менеджер, дедлайн...
        |
        v
Отправляет сообщение в Telegram-группу через бота
        |
        v
Сохраняет сообщение в историю чата
```

## Первоначальная настройка (для Директора)

Настройка выполняется один раз на сервере. Без неё письма и Telegram не работают.

### Шаг 1 — Настроить email (SMTP)

Для отправки писем-приглашений нужен почтовый ящик. В конфигурации сервера укажите:

| Параметр | Пример значения | Описание |
|---------|----------------|---------|
| SMTP_HOST | smtp.mail.ru | Адрес почтового сервера |
| SMTP_PORT | 465 | Порт (465 — SSL, 587 — STARTTLS) |
| SMTP_USERNAME | crm@festivalcolor.ru | Адрес отправителя |
| SMTP_PASSWORD | (пароль ящика) | Пароль от почты |
| SMTP_USE_TLS | true | Использовать шифрование |
| SMTP_FROM_NAME | Festival Color CRM | Имя отправителя в письме |

### Шаг 2 — Подключить Telegram-бот

В конфигурации сервера:

| Параметр | Описание |
|---------|---------|
| TELEGRAM_BOT_TOKEN | Токен бота, полученный от @BotFather в Telegram |

После указания токена бот автоматически начинает принимать команды от сотрудников.

> Если у вас ещё нет бота — напишите @BotFather в Telegram, команда `/newbot`, следуйте инструкциям. Скопируйте полученный токен.

## Подключение Telegram сотрудника

### Отправить приглашение

**Делает Директор (Руководитель студии):**

1. Открыть CRM → раздел **Сотрудники** (доступен только пользователям с правом на редактирование сотрудников)
2. Найти нужного сотрудника в таблице
3. В столбце **Действия** найти кнопку приглашения — последняя иконка в строке:
   - **Серая, неактивная** — у сотрудника не заполнен email (сначала внесите email в карточку)
   - **Синяя (иконка "человек со значком +")** — кнопка активна, нажмите для отправки приглашения
   - **Зелёная (иконка "человек с галочкой")** — Telegram уже подключён
4. Нажать синюю кнопку — система сгенерирует одноразовую ссылку (действует 7 дней) и отправит письмо

### Что получает сотрудник

Сотрудник получит письмо с темой **"Добро пожаловать в Festival Color CRM"**, которое содержит:
- Логин для входа в CRM
- Кнопку **"Подключить Telegram"**

### Как сотрудник подключает Telegram

1. Открыть письмо на любом устройстве
2. Нажать кнопку **"Подключить Telegram"**
3. Откроется Telegram с ботом CRM — нажать **"Старт"** или **"/start"**
4. Бот пришлёт подтверждение: *"Отлично, [Имя]! Ваш аккаунт привязан..."*

После этого в окне настроек уведомлений статус изменится на **"Telegram подключён"** (зелёный).

### Что делать, если ссылка устарела

Отправьте приглашение повторно — нажмите кнопку "Пригласить" ещё раз. Предыдущий токен станет недействительным, создастся новый.

## Настройка типов уведомлений

### Где находятся настройки

Настройки уведомлений доступны **любому сотруднику** — нажмите кнопку **"Настройки уведомлений"** (с иконкой шестерёнки) в верхней строке программы, рядом с именем пользователя. Откроется диалоговое окно настроек.

> Администрирование доступно только Директору и Руководителю студии. Настройки уведомлений специально вынесены в отдельное место, чтобы любой сотрудник мог управлять своими уведомлениями без доступа к разделу администрирования.

### Что видит Директор (Руководитель студии)

- Выпадающий список со всеми активными сотрудниками
- Полный набор настроек для каждого выбранного сотрудника
- Кнопку **"Тест уведомления"** (отправляет тестовое сообщение в Telegram)

### Что видит обычный сотрудник

- Только свои настройки (без выбора другого сотрудника)
- Те же галочки для управления своими уведомлениями

### Описание настроек

| Настройка | По умолчанию | Описание |
|-----------|-------------|----------|
| Включить Telegram уведомления | Вкл | Главный рубильник. Если выключен — ничего не приходит |
| Смена стадии CRM | Вкл | Уведомлять когда карточка переходит в новую колонку |
| Назначение исполнителем | Вкл | Уведомлять когда назначают ответственным за стадию |
| Предупреждение о дедлайне | Вкл | Уведомлять о приближении срока |
| Создание оплаты | Выкл | Уведомлять о новых записях оплаты *(отображается только если у сотрудника есть право доступа к оплатам)* |
| Новый надзор | Выкл | Уведомлять о событиях надзора |

### Как сохранить

После изменения галочек нажать кнопку **"Сохранить"** (жёлтая кнопка). Настройки применяются немедленно.

### Как проверить, что всё работает

1. Убедиться что статус показывает **"Telegram подключён"** (зелёный)
2. Нажать кнопку **"Тест уведомления"**
3. В Telegram сотрудника должно прийти сообщение: *"Тестовое уведомление — Система уведомлений CRM Festival Color работает корректно"*

## Приглашение клиента в чат

### Предварительные условия

Перед отправкой приглашения необходимо:
- В карточке CRM создан групповой чат (кнопка "Создать чат")
- У клиента заполнен email в карточке договора

### Как создать чат для карточки

1. Открыть карточку CRM (двойной клик по карточке)
2. В правой части диалога найти блок **"Групповой чат"**
3. Нажать кнопку **"Создать чат"**
4. Система создаёт Telegram-группу и сохраняет ссылку-приглашение

### Как пригласить клиента

1. После создания чата в том же блоке станет активной синяя кнопка с иконкой **Telegram** (без текста, подсказка при наведении: *"Пригласить клиента — отправить email со ссылкой на чат"*)
2. Нажать кнопку
3. Клиент получит письмо с темой **"Приглашение в проектный чат: [адрес объекта]"**

### Что получает клиент

Письмо содержит:
- Приветствие с именем клиента
- Карточку проекта: адрес объекта, тип проекта, имя менеджера
- Кнопку **"Присоединиться к чату"**

После нажатия кнопки клиент попадает в Telegram-группу проекта.

### Почему кнопка неактивна

Кнопка "Пригласить клиента" активна только если выполнены все условия:
- Чат создан
- У чата есть ссылка-приглашение (invite_link)
- У карточки есть привязанный договор
- У договора есть клиент с заполненным email

## Автоматические скрипты

### Что это такое

Скрипты — это шаблоны сообщений, которые система автоматически отправляет в групповой чат при наступлении события. Тексты настраиваются в разделе **Мессенджер → Шаблоны**.

### Доступные переменные в шаблонах

| Переменная | Что подставляется |
|-----------|-----------------|
| `{{client_name}}` | Полное имя клиента |
| `{{address}}` | Адрес объекта из договора |
| `{{city}}` | Город объекта |
| `{{contract_number}}` | Номер договора |
| `{{area}}` | Площадь объекта |
| `{{stage_name}}` | Название текущей стадии |
| `{{manager_name}}` | Имя менеджера |
| `{{senior_manager}}` | Имя старшего менеджера |
| `{{deadline}}` | Дата дедлайна по стадии |

### Пример шаблона

```
Добрый день, {{client_name}}!

По вашему проекту {{address}} завершён этап "{{stage_name}}".

Следующий шаг уже начат. Если возникнут вопросы — пишите в этот чат.

С уважением,
{{manager_name}}
Festival Color
```

## Частые вопросы

**Уведомления не приходят в Telegram**
- Нажмите кнопку "Настройки уведомлений" (шестерёнка) в верхней строке → проверьте статус ("Telegram подключён" зелёным?)
- Если нет — обратитесь к Директору, он отправит приглашение повторно
- Если да — проверьте, включена ли галочка "Включить Telegram уведомления"
- Директор может нажать "Тест уведомления" — если тест не пришёл, проблема с токеном бота на сервере

**Кнопка "Пригласить" у сотрудника недоступна**
- Проверьте, заполнен ли email в карточке сотрудника

**Кнопка "Пригласить клиента" неактивна**
- Создан ли чат для карточки?
- Заполнен ли email у клиента в карточке договора?

**Клиент не получил письмо**
- Письмо могло попасть в спам — попросите клиента проверить папку "Спам"
- Проверьте правильность email в карточке договора
- Убедитесь, что SMTP-сервис настроен (системный администратор)

**Автоматические сообщения не отправляются в группу**
- Создан ли чат для этой карточки?
- Настроены ли шаблоны-скрипты в разделе Мессенджер?
- Включён ли нужный скрипт (переключатель "Включено")?
